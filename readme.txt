= What's New in WPBookingly 1.3.1 =
Version 1.3.1 brings the service editor, checkout, customer account area, recurring-booking controls, and security model together into one consistent workflow.

**Two complete checkout modes**:
Choose WooCommerce mode to use WooCommerce taxes, checkout fields, coupons, terms, payment gateways, emails, and orders. The complete WooCommerce checkout opens inside the booking drawer, keeps the customer's selections while it loads, supports external gateway redirects, and retries temporary checkout failures without forcing the customer to start again. Custom/native mode provides WPBookingly's own cart, checkout, order confirmation, and customer account experience.

**Modern customer account centre**:
Custom/native payment mode automatically creates a WPBookingly My Account page with a responsive dashboard, recent orders, complete order details, billing information, profile settings, and password controls. WooCommerce mode adds booking actions to the standard WooCommerce My Account orders screen while preserving WooCommerce's native account and order flow.

**Customer booking management**:
Customers can view booking details, reorder an available service, request cancellation, and reschedule eligible appointments. Recurring orders include a dedicated series view where customers can edit an appointment, add another appointment, review paid and outstanding totals, or submit one cancellation request for the full series. Store policies and lead-time rules remain enforced.

**Professional service and booking screens**:
The service page, booking drawer, checkout confirmation, order-received details, account tables, action buttons, and reschedule dialogs now share a modern responsive design. Service Details retain the paragraphs, headings, lists, and formatting entered in the WordPress editor.

**Improved administration**:
The modern service editor covers categories and services, extra services, pricing, schedules, staff, recurring rules, taxes, waiting lists, FAQs, galleries, and formatted service details. The service list, booking administration, staff dashboard, reviews, cancellation requests, analytics, quick setup, and built-in help guide have also been improved.

**One-Click Business Templates**:
Start from the Service List and choose a ready-made business type to create a complete, ready-to-book service in seconds. A template can populate the service title, categories, services and prices, extras, features, overview, details, FAQs, operating schedule, capacity, recurring settings, sample staff profiles, and sample reviews. The imported service opens in the normal editor and every setting remains fully editable, so administrators can replace the sample content and publish the exact workflow their business needs.

**Advanced booking coupons**:
Create fixed, percentage, or fixed-price booking discounts with service, total, quantity, date, time, customer, account, staff, and usage restrictions. Coupon validation is shared by custom/native checkout, classic WooCommerce checkout, and WooCommerce Checkout Blocks so the same rule is applied on every supported checkout path.

**Reliability, privacy, and security**:
Checkout requests now use stricter server-side validation, protected booking access, payment reference verification, safer file uploads, capability and ownership checks, nonces, and rate limits. GDPR consent and customer data tools are integrated into the applicable checkout and account workflows.

= Booking and Account Shortcodes =
Use the following shortcodes when you need to place WPBookingly screens on custom pages:

* `[service-booking post_id="123"]` - display the booking interface for service post ID 123.
* `[mpwpb-user-dashboard]` - display the customer booking dashboard.
* `[custom_payment_my_account]` - display the custom/native payment My Account area. WPBookingly creates this page automatically when required.
* `[mpwpb_booking_confirmation]` - display a protected custom/native order confirmation on the confirmation page selected in Payment Settings.

== Installation ==

1. Upload the `service-booking-manager` folder to `/wp-content/plugins/`, or install WPBookingly from Plugins > Add New.
2. Activate WPBookingly through the Plugins screen. Activation registers the service post type and refreshes permalink rules automatically.
3. Open WPBookingly > Quick Setup and choose the payment mode that fits the site:
   * **Custom/native mode (WooCommerce not required):** WPBookingly provides its own cart, checkout, orders, confirmation screen, payment settings, and customer My Account page.
   * **WooCommerce mode (WooCommerce required):** Install and activate WooCommerce to use its taxes, checkout fields, coupons, gateways, emails, order management, and My Account pages.
4. Configure business hours, payment settings, booking policies, and customer account pages under WPBookingly > Settings.
5. Create a service, add its service items and schedule, then publish it.
6. Use the service permalink or `[service-booking post_id="123"]` on a custom page.

== Frequently Asked Questions ==

= Is the basic WPBookingly plugin free? =
Yes. The core plugin is free to use. WPBookingly Pro adds premium features and integrations for businesses that need a more advanced setup.

= Why does a service URL show a 404 error? =
Current versions register the service post type and refresh rewrite rules automatically during activation. If a site was upgraded from an older version, deactivate and reactivate WPBookingly once or open Settings > Permalinks and click Save Changes. If the URL still returns 404, verify the web server's WordPress rewrite rules and permissions.

= What shortcode displays a booking form? =
Use `[service-booking post_id="123"]` and replace `123` with the service post ID. The previous `[bookingplus]` example is obsolete.

= How do WooCommerce and Custom/native payment modes differ? =
WooCommerce is optional. WooCommerce mode uses the site's WooCommerce checkout, taxes, coupons, terms, gateways, emails, and orders. Custom/native mode works without WooCommerce and uses WPBookingly's own cart, checkout, confirmation, orders, payment settings, and My Account page. Choose one under WPBookingly > Settings > Payment Settings; the plugin does not replace the selected mode during checkout.

= Can WPBookingly accept bookings without WooCommerce? =
Yes. Select Custom/native mode in WPBookingly > Settings > Payment Settings. WPBookingly will handle the booking cart, checkout, native order, confirmation, and customer account workflow without WooCommerce. Available native gateways depend on the installed WPBookingly edition and the gateways enabled in Payment Settings.

= Which payment gateways can I use? =
In WooCommerce mode, WPBookingly uses the gateways enabled in WooCommerce. Custom/native mode provides its own enabled payment choices according to the installed WPBookingly edition and Payment Settings. Always test gateway credentials and HTTPS before accepting live payments.

= Where can customers manage their bookings? =
WooCommerce-mode bookings appear in WooCommerce My Account > Orders with the applicable WPBookingly actions. Custom/native bookings appear in the automatically created WPBookingly My Account page. You can also place `[mpwpb-user-dashboard]` on a protected customer page.

= Can customers cancel or reschedule a booking? =
Eligible bookings show cancellation and reschedule controls in the customer account area and supported order views. Cancellation is a request that remains pending until an administrator reviews it. Availability, booking status, ownership, and the configured cancellation or reschedule lead time are checked on the server.

= Can customers manage recurring appointment series? =
Yes, when recurring booking is enabled for the service and available in the installed edition. Customers can review the series, edit eligible appointments, add an appointment, and request cancellation of the complete series from the supported account view.

= What should I do if checkout takes longer to load? =
WPBookingly keeps the selected booking data and automatically retries temporary WooCommerce checkout-fragment failures. Confirm that the Cart, Checkout, and My Account pages are assigned in WooCommerce, at least one gateway is available, and caching or optimisation tools do not cache cart, checkout, account, or WordPress AJAX responses.

= Can I create custom services with WPBookingly? =
Yes. You can create services with their own dates, schedules, duration, prices, availability, service groups, extra services, staff, FAQs, and service details.

= How does the booking calendar work? =
The calendar and schedule controls use the service's working days, operating times, duration, capacity, holidays, off dates, staff availability, existing bookings, and booking rules to calculate the dates and time slots a customer can select.

= Where do I report security bugs found in this plugin? =
Please report security bugs found in the source code of the WpBookingly plugin through the [Patchstack Vulnerability Disclosure Program](https://patchstack.com/database/vdp/9e5fc652-d645-44ea-8d5a-0947db247fec). The Patchstack team will assist you with verification, CVE assignment, and notify the developers of this plugin.
